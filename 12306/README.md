# 小鲁班抢票助手 (12306 Intelligent Ticket Assistant)

一款基于Electron + Vue 3开发的12306智能抢票桌面应用，帮助用户在节假日自动抢购心仪的车票。

## 功能特性

### 核心功能
- ✅ **智能抢票**：自动完成12306购票流程，支持定时抢票
- ✅ **多任务管理**：支持创建多个抢票任务，批量管理
- ✅ **候补功能**：无票时自动提交候补订单
- ✅ **实时通知**：支持桌面通知、邮件通知
- ✅ **断点续传**：任务状态持久化，崩溃后自动恢复
- ✅ **验证码处理**：支持手动输入和OCR识别（预留接口）

### 技术特性
- 🚀 **高性能**：毫秒级响应，精准触发抢票请求
- 🔒 **安全可靠**：账号信息本地加密存储，不上传云端
- 🎨 **现代UI**：基于Element Plus的现代化界面
- 📦 **跨平台**：支持Windows 10/11和macOS
- 🔧 **一键启动**：自动检查并安装依赖

## 技术栈

- **前端框架**：Vue 3 + TypeScript
- **UI组件库**：Element Plus
- **桌面框架**：Electron 28
- **状态管理**：Pinia
- **路由管理**：Vue Router 4
- **数据库**：SQLite (better-sqlite3)
- **自动化工具**：Puppeteer
- **日志系统**：Winston
- **构建工具**：Vite 5
- **打包工具**：Electron Builder

## 使用说明

### 1. 创建抢票任务

1. 点击"新建任务"按钮
2. 填写任务信息：
   - 任务名称
   - 出发地、目的地
   - 出发日期
   - 座位类型（可多选）
   - 是否启用候补
3. 点击"确定"保存任务

### 2. 启动任务

1. 在任务列表中找到要执行的任务
2. 点击"启动"按钮
3. 系统将自动执行抢票流程

### 3. 查看日志

- 点击左侧菜单"系统日志"
- 可按级别、时间范围筛选日志
- 支持关键词搜索

### 4. 系统设置

- 点击左侧菜单"系统设置"
- 配置通知渠道（桌面、邮件）
- 调整抢票策略参数
- 设置验证码处理方式

## 配置说明

### 通知配置

- **桌面通知**：使用系统通知API
- **邮件通知**：需配置SMTP服务器信息

### 抢票策略

- **重试次数**：失败后重试的次数（1-10次）
- **延迟范围**：请求之间的随机延迟（避免被风控）
- **查询间隔**：查询余票的间隔时间
- **候补功能**：无票时自动提交候补订单

### 验证码处理

- **手动输入**：弹出验证码窗口，等待用户输入
- **OCR识别**：自动识别验证码（需配置OCR服务）

## 安全说明

- 所有账号信息使用AES-256加密存储在本地
- 密钥文件权限设置为600（仅用户可读写）
- 不上传任何用户数据到云端
- 所有网络请求仅与12306官网通信

## 常见问题

### Q: 为什么无法登录12306？
A: 请检查账号密码是否正确，网络是否正常。如遇到验证码，请手动输入。

### Q: 抢票失败怎么办？
A: 查看日志了解失败原因，可能是无票、网络问题或被风控。建议调整延迟参数。

### Q: 如何提高抢票成功率？
A: 
1. 提前设置好任务，在开售时刻准时启动
2. 启用候补功能
3. 合理设置延迟参数，避免被风控
4. 确保网络稳定
